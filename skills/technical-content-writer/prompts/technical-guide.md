# Technical Guide

<instructions>
Draft a long-form technical guide (blog or integration guide).
</instructions>

<process>
1. Confirm: scope (problem space vs single workflow), expected reader
   action (learn / try / implement).
2. Structure: problem framing → prerequisites → step-by-step (with
   working code samples) → troubleshooting → next steps → references.
3. Code samples: language version pinned, environment named, output
   shown after each step.
4. Trade-offs section: name when not to use this approach.
5. Write into `work/{brief_id}/v{n}.md`.
</process>
