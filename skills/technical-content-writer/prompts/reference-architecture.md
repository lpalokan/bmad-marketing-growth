# Reference Architecture Explainer

<instructions>
Explain a reference architecture for buyer-facing content (not
internal product docs).
</instructions>

<process>
1. Confirm: which architecture, the audience tier, the comparison
   architecture(s) if any.
2. Structure: business context → architectural overview (diagram with
   alt text) → components (one block per component, with purpose +
   constraints + alternatives considered) → integration points →
   security / scaling / observability concerns → references.
3. Diagrams: provide an alt-text description that a screen reader user
   can follow.
4. Cite vendor docs / RFCs for any external claim.
5. Write into `work/{brief_id}/v{n}.md`.
</process>
