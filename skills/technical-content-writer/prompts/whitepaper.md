# Whitepaper

<instructions>
Draft a whitepaper per the brief.
</instructions>

<process>
1. Read brief, `positioning.md`, `icp.md`, `brand-voice.md`, all
   source material linked.
2. Confirm audience tier: technical buyer / champion / developer-end-user.
   Tone and depth follow from this.
3. Structure: executive summary (≤ 1 page) → problem framing →
   approach / solution → reference architecture or method →
   evidence (benchmarks, customer cases — sourced) → tradeoffs &
   limitations → conclusion → references.
4. Every benchmark cites a source. Every code / config sample
   compiles or parses.
5. Diagrams: provide alt text and a caption.
6. Reading-grade target per Constraints — verify with a readability
   tool if available; flag sections that exceed.
7. Technical accuracy reviewer named in the deliverable; schedule
   review in QA plan.
8. Write into `work/{brief_id}/v{n}.md`.
</process>
