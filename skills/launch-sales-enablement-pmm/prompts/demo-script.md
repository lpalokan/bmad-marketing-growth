# Demo Script

<instructions>
Produce a timed demo script tied to the launch narrative.
</instructions>

<process>
1. Ask for: demo length (5 / 10 / 20 min), audience (technical /
   business / mixed), and the one feature/flow the demo must absolutely
   land.
2. Structure with timing markers:
   - 00:00 — Opener (tie to one-line story).
   - 00:30 — Problem framing (use a pain from `icp.md`).
   - 02:00 — Solution walkthrough (the must-land feature first).
   - {N}:00 — Recap + CTA.
3. Annotate each section with: SE talking points, common questions to
   expect, "show this if asked about X".
4. Output as `work/{brief_id}/v{n}-demo-script.md`.
</process>
