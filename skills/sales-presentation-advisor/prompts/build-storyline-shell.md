# Build a Storyline Shell (BD)

<instructions>
Generate an Act I / Act II / Act III scaffolding from audience +
desired decision when no source artifacts are available. Lighter than
BP; produces the structure for the user to fill in.
</instructions>

<process>

## 1. Intake (single numbered list, up to three questions)

1. **Audience** — who is in the room? (Cast them as protagonist.)
2. **Desired decision / outcome** — what must they decide or do?
3. **Length** — 5 / 15 / 30 / 45+ minutes.

Then separately ask depth (L1 storyline-only / L2 +notes / L3 +bullets).

## 2. Produce the shell

Write into `{output_folder}/work/{presentation-id}/v{n}.md`. The
shell uses the same structure as BP but with placeholders rather than
inherited content:

```
## Act I — Setup

### Slide 1 — Hook
**Headline:** [DRAFT — needs the "why now" trigger. Suggested
phrasing: "<one-sentence trigger relevant to audience>"]

### Slide 2 — Relevance
**Headline:** [DRAFT — audience's stake. Suggested phrasing:
"<audience role> face <change/pressure> that <consequence>"]

### Slide 3 — Challenge (Point A)
**Headline:** [DRAFT — current pain, named specifically and owned by
the audience]

### Slide 4 — Desire (Point B)
**Headline:** [DRAFT — desired outcome in the audience's words]

### Slide 5 — Map
**Headline:** [DRAFT — "Three things will get us from A to B:
<anchor 1>, <anchor 2>, <anchor 3>."]

## Act II — Develop

### Anchor 1: [DRAFT — most important takeaway as a sentence]
### Anchor 2: [DRAFT — second most important]
### Anchor 3: [DRAFT — third]

## Act III — Resolve

### Decision slide
**Headline:** [DRAFT — restate the desired decision]

### Next-step slide
**Headline:** [DRAFT — concrete action and owner]
```

## 3. Coach the user through filling it in

Don't fabricate anchors. Instead:

- For each slot, ask one focused question to elicit the user's input.
- When the user offers content, rewrite it as a complete-sentence
  headline in active voice.
- After each act is filled, run a mini-check: "Does this Challenge
  contrast cleanly with this Desire?" "Does this Anchor support that
  Desire?"

## 4. Hand-back

Once the shell is filled (even with `[NEEDS INPUT]` markers), tell
the user:

- File written.
- Open slots count and what each one needs.
- Suggested next step: either run BP if source artifacts now exist,
  or HD on the headlines they did fill in.

</process>
