# Tension Check (TX)

<instructions>
Focused diagnostic on the Challenge ↔ Desire gap. Tension between the
two is the engine of the storyline; if it collapses, the audience
disengages.
</instructions>

<process>

## 1. Extract A and B

Ask the user to state, in their own words:

- **A — Challenge:** the unresolved state of affairs the audience
  currently faces.
- **B — Desire:** the outcome they want to be in.

If the user has a draft storyline, pull A from the Challenge slide and
B from the Desire slide and quote them back for confirmation.

## 2. Diagnose

Score the A↔B pair against these tests. Mark each PASS / FAIL with a
one-sentence reason.

| Test                                                                                   | PASS / FAIL | Reason |
|----------------------------------------------------------------------------------------|-------------|--------|
| A is concrete and the audience owns it (not someone else's problem)                    |             |        |
| B is concrete and visualisable (not vague aspiration like "be successful")             |             |        |
| The gap between A and B is **real** — there's actual work to do to cross it            |             |        |
| The audience cares about closing the gap (relevance ties to their KPIs / role)         |             |        |
| A and B are stated in the **same dimension** (e.g. both about cycle time, both about cost) |             |        |
| Neither A nor B name the presenter's product (the product is a vehicle, not the goal)  |             |        |

## 3. Common failure modes (call out by name when they appear)

- **Vague desire** — "We want to grow" → no shape. Sharpen: "Convert
  one inbound demo per AE per day by Q4."
- **Audience-misaligned challenge** — "Our team struggles to ship
  fast" delivered to a CFO → wrong owner of the problem. Reframe to
  the CFO's pain (cost overruns, missed forecasts).
- **Collapsed tension** — A and B differ only by a word ("Slow
  pipeline" → "Faster pipeline"). The audience can't see the work
  between them. Add specificity to one or both.
- **Product-as-protagonist** — A is about the audience, B is about
  the product launching. Demote the product to a tool in Act II.
- **Cross-dimensional A↔B** — A is about cost, B is about morale.
  Pick one dimension and stay with it; defer the other to a
  guardrail line.

## 4. Rewrite (offer a sharper A↔B pair)

Propose 1–2 alternative framings. Show the move you made:

```
Original:
  A: "Sales reps waste time on manual CRM work."
  B: "We need better tooling."
Issue: B doesn't name an outcome; product-as-protagonist.

Sharper:
  A: "AEs lose 6 hours per week to CRM workarounds, slipping 1 in 5
      forecasted deals."
  B: "Each AE recovers a full selling day per week, lifting Q4 forecast
      attainment by 8 points."
Move: gave A a unit (hours, deals) and gave B the same unit
      (selling day, forecast points). Tension is now visible.
```

## 5. Hand-back

State the verdict in one sentence: "The current A↔B holds" or "The
current A↔B collapses — see the sharper framing." Suggest the next
step: re-run SR on Act I with the sharper framing, or push the new A↔B
back into the storyline file directly.

</process>
