# Sales Enablement

<instructions>
Produce a sales enablement bundle for a launch or campaign: battlecard,
one-pager, demo script, FAQ. Delegates to launch-sales-enablement PMM.
</instructions>

<process>
1. Confirm scope with the user: which deliverables they actually need
   today (don't generate everything by default).
2. Load `positioning.md`, `icp.md`, the relevant competitive teardown
   (from `work/` if recent), and `brand-voice.md`.
3. Issue a brief to `launch-sales-enablement-pmm` using
   `brief-templates/launch-enablement.md`. Acceptance criteria
   include: each asset names the persona it's for; the demo script
   labels timing per section; the FAQ covers the top 5 objections from
   the latest competitive teardown.
4. Review per `docs/protocol.md`. On APPROVED, ask the user where
   final assets go in their sales-collateral repo (do not invent a
   location).
</process>
