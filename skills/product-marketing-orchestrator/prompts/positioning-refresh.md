# Positioning Refresh

<instructions>
Refresh (or initialise) `company-context/positioning.md`.
</instructions>

<process>
1. Read the current `positioning.md` (if it exists) and `icp.md`
   (required — refuse to proceed without it; refer the user to PI).
2. Walk through the positioning template, asking one question at a time:
   - For (target segment) who (need or problem),
   - (product) is a (category)
   - that (key differentiated benefit),
   - unlike (primary alternative — name 2-3 by name),
   - because (reason to believe — specific, defensible, not "AI").
3. Build the value pyramid: functional benefit → emotional benefit →
   social/identity benefit. Each layer must be specific to the ICP.
4. Sanity check against the ICP: would the named buyer persona actually
   recognise this positioning as describing their problem? If not, push
   back.
5. Draft `positioning.md`. Required sections: Positioning Statement,
   Category, Target, Primary Alternatives, Value Pyramid, Reasons To
   Believe (cite sources), Anti-positioning (what we explicitly are
   NOT).
6. Write with full frontmatter (see icp-refresh.md). Confirm path.
</process>
