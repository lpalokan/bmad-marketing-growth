# Messaging Pillars

<instructions>
Produce three messaging pillars plus proof points, derived from
positioning. Output is consumed by the Positioning & Messaging PMM via
a brief.
</instructions>

<process>
1. Load `positioning.md` (required) and `icp.md` (required).
2. Issue a brief to `positioning-messaging-pmm` using
   `brief-templates/positioning.md`. Acceptance criteria MUST include:
   - exactly 3 pillars,
   - each pillar maps to one of the value-pyramid layers,
   - each pillar has 3+ proof points with sources,
   - vocabulary lines up with `brand-voice.md`,
   - no pillar invents a competitor claim.
3. Wait for `v1.md`. Review against the brief checklist per
   `docs/protocol.md`. Issue `v1-review.md` with one of: APPROVED,
   NEEDS_REVISION (numbered required changes), ESCALATED.
4. On APPROVED, update `company-context/positioning.md` Messaging
   Pillars section with the approved pillars (ask user first), then
   freeze the work folder.
</process>
