# Contingency Plan

<instructions>
Build the contingency plan for the event.
</instructions>

<process>
1. Speaker drop: backup speaker named, swap protocol.
2. AV failure: webinar — backup platform; in-person — backup hardware,
   on-site engineer.
3. Low show-up: pre-recorded backup, on-demand pivot plan.
4. Communication tree: who tells whom in each scenario.
5. Write into `v{n}.md`.
</process>
