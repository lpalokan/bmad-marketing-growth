# Schema Markup Spec

<instructions>
Spec the schema.org markup for a content type.
</instructions>

<process>
1. Identify the content type (Article, Product, FAQPage, HowTo,
   Organization, Event, Webinar, etc.). Verify on schema.org that
   the type still exists and is the right fit.
2. Required properties + recommended properties for the type.
3. JSON-LD example (illustrative — do not invent numbers for
   aggregateRating without source).
4. Validation: links to Rich Results Test and Schema Markup Validator
   for QA.
5. Implementation notes: where it lives (template / per-page),
   server-side vs client-side concerns for indexation.
6. Write into `v{n}.md`.
</process>
