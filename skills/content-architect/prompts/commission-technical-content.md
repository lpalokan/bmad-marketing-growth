# Commission Technical-Content Brief

<instructions>
Commission a technical-content piece. Delegates to
technical-content-writer (Theo).
</instructions>

<process>
1. Confirm: deliverable type (whitepaper, solution brief, technical
   guide, architecture diagram explainer), target audience (technical
   buyer / champion / end-user developer), length, deadline,
   distribution.
2. Issue brief using `brief-templates/technical-content.md`. Acceptance
   criteria include: technical accuracy review path, source citations
   for any benchmark, code / config samples that compile / parse,
   accessibility (alt text for diagrams).
3. Review per `docs/protocol.md`.
</process>
