# Glossary

<instructions>
Build or refresh the glossary.
</instructions>

<process>
1. Pull terms from: product names, category nouns from
   `positioning.md`, regulated terms (per industry), recurring
   campaign phrases.
2. Per term: source-language entry, allowed translations per locale,
   forbidden translations (with reason), notes for translators.
3. Versioning: track glossary version in frontmatter.
4. Write into `v{n}.md`.
</process>
