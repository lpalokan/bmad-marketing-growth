# Attribution Plan (Double-Count-Aware)

<instructions>
Build an attribution plan that does not double-count with other
channels.
</instructions>

<process>
1. Inventory other channels claiming credit for these accounts: paid
   search, paid social/demand, content, lifecycle.
2. Define the credit rule: each opportunity gets ONE primary source;
   secondary sources get "assist" credit (not pipeline-dollar credit).
3. Choose attribution method: pre-post lift on named accounts (best
   for ABM), MMM (if MMM exists), or shared-source attribution table.
4. Reporting: dashboard URL, day-of-week, named owner.
5. Recommend Pixel Metrics validate the model before launch.
6. Write into `v{n}.md`.
</process>
