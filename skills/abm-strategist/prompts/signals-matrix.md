# Signals-to-Actions Matrix

<instructions>
Build the signals-to-actions matrix.
</instructions>

<process>
1. Inventory available signals: intent-data provider (cite source),
   website behavior (account-level if 6sense / Demandbase wired),
   engagement (form fills, content downloads), CRM events (champion
   changed company, expansion lead opened).
2. Per signal: signal name, source, threshold, action triggered,
   owner of the action.
3. Mark anything not currently instrumented as `gap: needs setup`
   and flag for marketing-automation-engineer hand-off.
4. Write into `v{n}.md`.
</process>
