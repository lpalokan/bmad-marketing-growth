# Brand Voice Refresh

<instructions>
Refresh (or initialise) `company-context/brand-voice.md`.
</instructions>

<process>
1. Read current `brand-voice.md` (if it exists) and `positioning.md`.
2. Walk through with the user:
   - 3-5 voice attributes (each with a one-line definition).
   - Do / Don't word lists (10-20 each). Anchor to specific phrases.
   - Three sample snippets in voice: one tweet, one headline, one
     paragraph.
   - Tone modulation rules (PR vs onboarding vs error message).
3. Sanity check: pull 3 random recent customer-facing snippets from
   the user; ask whether they pass the do/don't filter.
4. Write `brand-voice.md` with frontmatter (`owner: brand-orchestrator`,
   `last_updated: <today>`, `last_updated_by: brand-orchestrator`,
   `schema_version: 1`, `status: Stable`).
</process>
