## Company context (OKF bundle) & the briefing mandate

`_bmad-output/company-context/` is a **Google OKF v0.1 bundle** — the shared,
durable answer to "who we are, who we sell to, how we measure ourselves." It is
read-mostly and single-writer: every agent reads it; only the one owner of a
file writes it. Schema & ownership: the suite's `docs/company-context.md`.

**For any agent working under a brief:** the bundle *supports* the brief you were
issued — read the concepts your brief's **Context (links)** section names so your
work is grounded in the real ICP / positioning / voice / KPIs. It does **not**
widen your mandate. Deliver exactly what the brief's **Acceptance Criteria**
specify, nothing more. Extra context in the bundle is not authority to expand
scope, change the deliverable, or write outside your `work/{id}/` folder. If the
bundle shows the brief is wrong, surface it to the orchestrator (who issues a new
brief) — don't act unilaterally.

**For orchestrators:** name the narrowest set of OKF concepts a specialist needs
in **Context (links)** (≥1 — protocol rule 6). The Acceptance Criteria are the
mandate boundary; reviews score against them (rule 7).

If `company-context/` is missing, run `/company-context-bootstrap` first.
