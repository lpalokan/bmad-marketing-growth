# Accessibility (WCAG 2.2 AA) Check

<instructions>
Run a focused accessibility check on a page or variant.
</instructions>

<process>
1. Cover at minimum: colour contrast, focus order, keyboard
   navigability, alt text, form labels, ARIA misuse, heading order.
2. Severity per finding: blocker / major / minor.
3. Cite the WCAG 2.2 success criterion per finding (e.g. "1.4.3
   Contrast (Minimum)").
4. Output a remediation list with owner and ETA. Mark which findings
   are launch-blockers.
5. Write into `v{n}.md`.
</process>
