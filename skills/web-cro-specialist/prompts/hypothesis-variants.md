# Hypothesis & Variants

<instructions>
Produce the hypothesis statement and the variants spec for a CRO test.
</instructions>

<process>
1. Read brief, `positioning.md`, `icp.md`, `brand-voice.md`.
2. Frame the hypothesis as: "If we change {X} (which currently does
   {Y}), we expect {primary metric} to move by {amount}, because
   {reasoning tied to ICP/positioning}."
3. Spec the variants:
   - Control (current state, screenshot or copy block).
   - Challenger(s): for each, list the deltas vs control and the
     specific copy / layout change.
4. Cap at 3 variants total. More requires explicit orchestrator approval.
5. Write into `v{n}.md`.
</process>
