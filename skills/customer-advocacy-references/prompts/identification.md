# Advocate Identification

<instructions>
Identify candidate advocates from CRM and product signals.
</instructions>

<process>
1. Ask the user for: CRM list of healthy accounts, NPS distribution
   (current), expansion / contraction status per account.
2. Score candidates: NPS ≥ 9 + tenure ≥ 12 months + no recent
   support escalation + CSM endorsement.
3. Tier: gold (cite-able publicly), silver (cite-able privately),
   bronze (helpful but not for marketing).
4. CSM partnership: CSM owns the relationship; Marketing requests
   participation via CSM, never direct outreach.
5. Consent flow: legal-reviewed template per region (GDPR / CASL /
   etc.).
6. Write into `v{n}.md`.
</process>
