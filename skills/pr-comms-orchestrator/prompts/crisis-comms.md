# Crisis Comms Playbook

<instructions>
Build or refresh the crisis-comms playbook.
</instructions>

<process>
1. Scenarios: security incident, outage, executive departure, customer
   data leak, regulatory event, public mistake.
2. Per scenario: trigger, decision-maker, internal-comms first,
   external-comms timeline, holding-statement template, spokesperson.
3. Approval chain: legal → CEO → CCO → publish.
4. Post-incident: review, learnings, playbook update.
5. Write to `work/{yyyy-crisis-playbook}/v{n}.md` (refresh annually).
</process>
