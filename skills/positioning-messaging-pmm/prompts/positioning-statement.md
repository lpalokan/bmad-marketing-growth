# Positioning Statement

<instructions>
Draft or revise a positioning statement against a brief. Output is part
of `v{n}.md` for the work folder; never overwrite `company-context/`.
</instructions>

<process>
1. Read the brief, `positioning.md` (current), `icp.md`, and any
   referenced competitive teardowns.
2. Use the canonical template:
   For (target segment) who (need/problem), (product) is a (category)
   that (key differentiated benefit), unlike (named alternatives),
   because (defensible reason to believe).
3. Each slot must reference a source from Context. Mark unsourced
   slots `[needs source]` rather than inventing.
4. Sanity check against the brief's Acceptance Criteria. If you can't
   satisfy a criterion with the inputs you have, flag it in the
   deliverable's "Open Questions" section — do not fabricate.
5. Write the section into `work/{brief_id}/v{n}.md`.
</process>
